  <action>
1. Install dependencies: `npm install p-retry p-throttle`

2. Create `src/tenable-client.ts` with a `TenableClient` class following the research document's Pattern 1. The class must:

**Constructor** accepts `TenableClientConfig`:
```typescript
export interface TenableClientConfig {
  baseUrl: string;        // e.g. "https://cloud.tenable.com"
  accessKey: string;
  secretKey: string;
  timeoutMs?: number;     // default 30_000
  maxRetries?: number;    // default 3
  rateLimitPerMin?: number; // default 100
}
```
- Store config with defaults filled in
- Create a p-throttle instance: limit = `Math.floor(rateLimitPerMin / 6)`, interval = `10_000` (10-second window) (HTTP-06)
- Create `throttledFetch` by wrapping native `fetch` with the throttle

**`request<T>(method: string, path: string, body?: unknown): Promise<T>`** (HTTP-01):
- Build URL: `${baseUrl}${path}`
- Wrap the fetch call in `pRetry()` with `retries: maxRetries`, `factor: 2`, `minTimeout: 1000` (HTTP-05)
- INSIDE the pRetry callback (per-attempt, not once):
  - Create `AbortSignal.timeout(this.config.timeoutMs)` -- new signal per attempt to avoid pitfall 1 from research (HTTP-03)
  - Call `this.throttledFetch(url, { method, headers, body, signal })`
  - Headers: `X-ApiKeys: accessKey=${accessKey};secretKey=${secretKey}` (no trailing semicolon, no space) + `Content-Type: application/json` + `Accept: application/json` (HTTP-02)
  - Body: `JSON.stringify(body)` if body provided, `undefined` otherwise
  - On non-OK response:
    - 429: throw a regular `Error` with statusCode property so p-retry retries it. Log retry-after header value to stderr (HTTP-05)
    - 400, 401, 403, 404, 409: throw `AbortError` from p-retry (non-retryable). Attach statusCode property.
    - 5xx: throw regular `Error` (retryable)
  - On OK response: `return (await response.json()) as T`
- After pRetry resolves/rejects, catch errors and convert:
  - If error has `.statusCode`, use `httpStatusToErrorType(statusCode)` to map and throw `createNessusError(type, message)`
  - If error is DOMException with name `TimeoutError`, throw `createNessusError(NessusErrorType.TIMEOUT, ...)`
  - If error is DOMException with name `AbortError`, throw `createNessusError(NessusErrorType.API_ERROR, 'Request aborted')`
  - Otherwise re-throw wrapped in `createNessusError(NessusErrorType.API_ERROR, ...)`

**IMPORTANT**: Import `AbortError` from `p-retry` (not native) for the "stop retrying" signal (pitfall 5 from research). Distinguish from native DOMException AbortError by checking `error.name === 'AbortError'` on DOMException instances.

**Convenience methods:**
- `get<T>(path: string): Promise<T>` -> calls `request<T>('GET', path)`
- `post<T>(path: string, body?: unknown): Promise<T>` -> calls `request<T>('POST', path, body)`

All local imports use `.js` extension. Export both `TenableClient` and `TenableClientConfig`.
