<task type="auto">
  <name>Task 2: Test TenableClient HTTP functions with mocked fetch</name>
  <files>tests/tenable-client.test.ts</files>
  <action>
Create `tests/tenable-client.test.ts` testing the TenableClient class with `vi.stubGlobal('fetch', mockFetch)`.

CRITICAL: Stub fetch BEFORE constructing the client (p-throttle captures the reference at construction time). Use `maxRetries: 0` and `rateLimitPerMin: 6000` in all tests to avoid slow tests from retries and throttling.

**Setup pattern (beforeEach):**
```typescript
let mockFetch: ReturnType<typeof vi.fn>;
let client: TenableClient;
beforeEach(() => {
  mockFetch = vi.fn();
  vi.stubGlobal('fetch', mockFetch);
  client = new TenableClient({
    baseUrl: 'https://cloud.tenable.com',
    accessKey: 'test-access',
    secretKey: 'test-secret',
    maxRetries: 0,
    rateLimitPerMin: 6000,
  });
});
```

**Test cases:**

1. **Auth header format:** Call `client.get('/test')` with mock returning 200. Verify `mockFetch` was called with URL `https://cloud.tenable.com/test` and headers containing `'X-ApiKeys': 'accessKey=test-access;secretKey=test-secret'`.

2. **GET request:** Mock returns `{ data: 'test' }` with status 200. Verify `client.get('/endpoint')` resolves to `{ data: 'test' }` and method is GET.

3. **POST request with body:** Mock returns 200. Call `client.post('/endpoint', { key: 'val' })`. Verify fetch was called with method POST, body is JSON-stringified `{ key: 'val' }`, and Content-Type header is `application/json`.

4. **HTTP 401 error mapping:** Mock returns 401. Verify `client.get('/test')` rejects. Catch the error and verify it contains "401" in message.

5. **HTTP 404 error mapping:** Mock returns 404. Verify `client.get('/test')` rejects with error containing "404".

6. **HTTP 429 error mapping:** Mock returns 429 with `retry-after: 60` header. Verify `client.get('/test')` rejects (since maxRetries: 0).

7. **HTTP 500 error mapping:** Mock returns 500. Verify `client.get('/test')` rejects.

8. **Timeout handling:** Mock fetch that never resolves (or use `AbortSignal.timeout`). Create a client with `timeoutMs: 100`. Verify the request rejects within a reasonable time. Note: the timeout is set via `AbortSignal.timeout(ms)` in the request method -- the mock fetch should check the signal. Simplest approach: mock fetch to return a delayed response and verify rejection. Use `vi.fn().mockImplementation(() => new Promise((resolve) => setTimeout(resolve, 5000)))` and set `timeoutMs: 50`.

9. **Base URL trailing slash normalization:** Construct client with `baseUrl: 'https://cloud.tenable.com/'`. Call `client.get('/test')`. Verify URL is `https://cloud.tenable.com/test` (no double slash).

For mock Response objects, use the native `new Response(JSON.stringify(body), { status, headers })` pattern.
