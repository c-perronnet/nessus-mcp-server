<task type="auto">
  <name>Task 1: Test all 7 tool handlers in mock mode</name>
  <files>tests/tool-handlers.test.ts</files>
  <action>
Create `tests/tool-handlers.test.ts`. Use `.js` extensions in imports.

Call `initializeNessusApi({ useMock: true })` in `beforeAll` -- this activates mock mode so handlers use mock-data.ts instead of real API. No fetch mocking needed.

**Test structure -- one describe block per tool:**

1. **list_scan_templates:**
   - Returns content array with type 'text'
   - Parsed text contains `templates` array
   - Templates have `title` and `uuid` fields
   - At least one template exists

2. **start_scan:**
   - Valid request `{ target: '192.168.1.1', scan_type: 'basic' }` returns content with `scan_id` and `status: 'queued'`
   - `isError` is undefined/falsy on success
   - Invalid input (empty target) returns `isError: true`

3. **get_scan_status:**
   - First call `startScanToolHandler({ target: '10.0.0.1', scan_type: 'basic' })` to create a scan and extract scan_id from response
   - Then call `getScanStatusToolHandler({ scan_id: extractedScanId })` and verify response has `status` field
   - Invalid scan_id returns `isError: true`

4. **get_scan_results:**
   - Use a scan_id from a started scan
   - Call `getScanResultsToolHandler({ scan_id: scanId })`
   - Response should have content (mock mode returns results immediately or an error -- either is valid, test for no crash)
   - Invalid scan_id returns `isError: true`

5. **list_scans:**
   - Returns content with type 'text'
   - Parsed text is valid JSON (either has scans array or is valid structure)

6. **search_vulnerabilities:**
   - Call with `{ keyword: 'sql' }` (mock data has SQL injection vuln)
   - Returns content, no crash
   - Parsed text has results array or vulnerabilities

7. **get_vulnerability_details:**
   - Call with `{ vuln_id: 'CVE-2021-44228' }` (Log4Shell -- likely in mock data) or another known mock vuln ID
   - Returns content, no crash
   - If mock data doesn't have that ID, test that `isError: true` is returned gracefully

**Pattern for all tests:**
```typescript
const result = await handlerFn(args);
expect(result.content).toBeDefined();
expect(result.content[0].type).toBe('text');
const parsed = JSON.parse(result.content[0].text);
// assert on parsed fields
```

For error cases, wrap handler call and check either `result.isError === true` or that it doesn't throw an unhandled exception (handlers should catch errors and return MCP error responses).
